API Information

Rate Limits

  • General Endpoints: 1000 requests per hour
  • Real-time Endpoints: 500 requests per hour
  • Telemetry Endpoints: 300 requests per hour
  • Burst Limit: 10 requests per second

Response Format

All API responses are in JSON format with consistent structure:

{ "data": { ... }, // Main response data "metadata": { // Request metadata "timestamp": "2025-07-29T09:30:00Z", "response_time_ms": 145, "cache_status": "hit" }, "session_info": { // Session context "year": 2024, "grand_prix": "Bahrain", "session": "Race" } }

Error Handling

Error responses include detailed information:

{ "error": "Missing required parameter: driver", "error_code": "MISSING_PARAMETER", "status": 400, "suggestion": "Add driver parameter: ?driver=VER" }

Data Freshness

  • Real-time Data: Updated every 1-2 seconds during live sessions
  • Historical Data: Cached for optimal performance
  • Analytics: Updated within 30 seconds of new data
  • Seasonal Data: Updated after each race weekend
